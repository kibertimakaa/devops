# Как использовать Git

## Основные команды Git
- git init — инициализация нового репозитория.
- git clone URL — клонировать существующий репозиторий.
- git status — показать текущий статус репозитория.
- git add файл — добавить файл в индекс для коммита.
- git commit -m "сообщение" — создать коммит с описанием изменений.
- git push — отправить изменения на удалённый репозиторий.
- git pull — получить обновления из удалённого репозитория и слить с локальной веткой.
- git branch — просмотр и управление ветками.
- git checkout ветка/коммит — переключиться на другую ветку или коммит.

## Типичный рабочий процесс

1. Клонировать репозиторий:

```
git clone https://github.com/user/repo.git
```

2. Создать свою ветку для задачи:

```
git checkout -b feature-branch
```

3. Работать с файлами, затем подготовить изменения к коммиту:

```
git add changed_file.py
```

4. Зафиксировать изменения с понятным комментарием:

```
git commit -m "Добавлен новый функционал"
```

5. Отправить ветку на сервер:

```
git push origin feature-branch
```

6. Создать pull request (merge request) через веб-интерфейс.

## Советы для эффективной работы с Git
- Используйте осмысленные сообщения коммитов.
- Часто коммитьте небольшие изменения.
- Регулярно обновляйте локальную ветку через git pull.
- Используйте ветки для разных задач и фич.
- Используйте .gitignore для исключения лишних файлов из репозитория.
